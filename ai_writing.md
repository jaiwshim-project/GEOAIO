## 핵심 공식

```
AI 인용 = EEAT × 데이터량 × 질문형 × 반복성 × 외부신호 × CTA
```

어느 한 요소가 0이면 AI 인용 가능성도 0 (곱셈).

## 6가지 원칙 (요약)

| 원칙 | 핵심 |
|---|---|
| 0. CEP 장면 점유 | 사람이 아닌 "순간" 표적 |
| 1. 작성자 권위 | 도입부+결론 별도 단락 |
| 2. 정의문 | 각 H2 첫 문장 "X는 ~이다" |
| 3. 구조화 | 번호 리스트·표·Q:A:·명시적 수치 |
| 4. AI 인용 공식 | 질문형 H2 + 반복 브랜드 + 케이스 + 외부신호 |
| 5. 행동 유도 | 핵심 답 + CTA 솔루션 + TL;DR |

## 7단계 골격 (의무)

1. 도입부
2. H2 5~7개 (5개 이상 질문형)
3. 단계별 프로세스 (번호 리스트)
4. 사례·수치
5. FAQ Q1·Q2·Q3+
6. 결론 + CTA
7. 비교 표 3열

## 분량 권장

| 용도 | 분량 |
|---|---|
| 짧은 답변 | 1,200~1,500자 |
| **표준 GEO 콘텐츠** | **1,800~2,200자** |
| 깊이 있는 가이드 | 2,200~2,500자 |

## 측정 4신호 목표

- 브랜드 반복 ≥ 50%
- 질문형 H2 ≥ 50% (이상 70%+)
- 외부 신호 ≥ 30%
- CTA 도달률 ≥ 50%

## 1개 글 → 30개 파생

마스터 글 1개를 30개 질문형 파생으로 분해.
- 마스터는 그대로 보존
- 파생이 AI 인용 담당
- 데이터량 부족 문제 해결

## 결정 트리

```
타겟이 사람? → 장면으로 변환
브랜드 1회? → 2회 이상
설명형 H2? → 질문형
짧은 결론 X? → 각 H2 끝 "핵심:"
케이스 X? → 4단 구조
외부 인용 X? → RAG에 후기·커뮤니티 키워드
CTA X? → "[X]는 [브랜드]로 해결"
1글로 끝? → 30개 파생
1500자 미만? → 1800자+
```
